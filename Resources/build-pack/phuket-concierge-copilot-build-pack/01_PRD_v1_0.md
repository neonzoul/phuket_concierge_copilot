# PRD v1.0, Build-Facing Summary

Source:
- `phuket_concierge_copilot_PRD_v1_0.md`

## Scope
Build the smallest demonstrable system for demo and 30-day pilot.

### In scope
- Guest web chat
- Verified knowledge answering
- Request capture
- Human handoff
- Staff inbox / dashboard
- Mitri Guest Brief
- Weekly Insight
- Demo seed/reset
- Append-only event logging

### Out of scope
- Native mobile app
- Full PMS integration
- Real-time inventory sync
- Payment, refund, cancellation automation
- Autonomous booking confirmation
- Medical, legal, or safety advice
- Multi-property administration
- Ten-language support

## Core Users
- Guest
- Front Desk / Guest Relations
- Wellness Team
- Operations Manager
- Owner / GM

## Required Functional Behavior
- Free-text English guest input
- Intent classification for every message
- Retrieval from verified knowledge only
- No unsupported factual claims
- No-guess behavior for unknown facts
- Pending request creation for confirm-state interactions
- Human handoff with summary, reason, priority, and team
- Request fields only when missing
- Final confirmation only after human confirmation
- Guest Brief updates from approved facts and staff actions
- All AI and human actions written to event log

## Data Entities

## Demo Success Criteria
- Known FAQ accuracy >= 95 percent on approved test set
- Unknown facts: 100 percent no-guess behavior
- Red cases: 100 percent human handoff
- No fake confirmation
- Request creation captures or asks for required fields
- Guest Brief readable in <= 20 seconds
- Owner report readable in <= 3 minutes
- Demo duration: 7 to 10 minutes

## Recommended Build Order
1. Demo data
2. Knowledge base
3. Risk classifier
4. Safe answer generator
5. Guest chat
6. Pre-arrival flow
7. Request and handoff services
8. Slack adapter
9. Dashboard
10. Guest Brief
11. Weekly Insight
12. Channel polish
